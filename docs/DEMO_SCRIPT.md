# Diana 90-second demo script and storyboard

This file is a script only. It does not claim that a video has been recorded or uploaded.

## 0:00–0:12 — The infrastructure gap

“Hormonal-health ML can look impressive while leaking one participant across train and test, using future cycle information, or treating consumer urinary readings as serum truth. Diana's primary contribution is Hormonbench—Challenge Layer 01 infrastructure that makes those mistakes fail.”

Show the `benchmark/` and `model/` boundary.

## 0:12–0:24 — The frozen benchmark

“For each day `t`, Hormonbench uses exactly 14 days of approved wearable summaries through `t` to forecast genuinely observed urinary LH, E3G, and PdG at `t+1`. Five participant folds test all 20 participants once. The primary metric gives every participant equal influence.”

Show the task card and `t-13...t -> t+1` diagram.

## 0:24–0:36 — Cold start and measurement burden

“Cold start exposes no personal hormone labels. A separate personalization track authorizes only the earliest zero, three, or seven complete readings and scores every budget on the same later suffix. That isolates the value—and burden—of personal tests.”

Show the K=0/3/7 common-suffix diagram and measurement-budget curve.

## 0:36–0:47 — Three classical baselines

“The comparison suite is deliberately small: participant-equal population median, wearable Ridge, and participant-weighted CatBoost. These are exactly three active classical families, all using the same frozen benchmark inputs.”

Show the aggregate cold-start leaderboard.

## 0:47–1:03 — Diana-H3P

“Diana-H3P is one Challenge Layer 02 reference—not a separate product. Layer 1 stacks the three population experts using participant-disjoint development predictions. Layer 2 updates a joint three-hormone personal posterior using only the authorized K measurements.”

Show population median + Ridge + CatBoost → wearable prior → personal posterior.

## 1:03–1:14 — Uncertainty and honest evidence

“The model emits 80% participant-block calibrated research intervals, not clinical confidence intervals. The canonical aggregate result is descriptive and post-hoc because earlier benchmark results were already inspected before H3P was designed; we do not claim untouched-test or clinical superiority.”

Show the verified uncertainty summary and exact canonical aggregate result generated by the runner.

## 1:14–1:24 — Reuse and privacy

“A licensed mcPHASES user can emit the stable prediction schema and run the evaluator without importing our models. Dataset rows, identities, truth, predictions, and calibration mappings remain private; only aggregate evidence is public.”

Show the evaluator command, explicit manifest, and privacy check.

## 1:24–1:30 — Limitation close

“Diana is not a diagnostic application. It is open benchmark infrastructure for asking more causal, reproducible, and measurement-aware hormonal-health research questions.”

End on Hormonbench as the reusable asset and link to the repository documentation.
